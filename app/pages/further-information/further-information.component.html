Why we need the .htaccess file in a production environment

When you deploy your build to a production server without an .htaccess file you will drive into an issue: As soon as you reload your app you'll get an 404 HTTP Code or an 500 HTTP Code because the server can not resolve your Angular routes. These routes are managed by Angular on the client-side and not on the server-side (anyway SEO for Angular generated pages do not work for the same reason. If you need SEO, see Angular Universal).

This issue only happens if you are using the PathLocationStrategy, because it adds a rout as an URI. Angular supports two different location strategies:

  • PathLocationStrategy — the default "HTML5 pushState" style. (example: http://example.com/page1)
  • HashLocationStrategy — the "hash URL" style. (example: http://example.com/#/page1)

The .htaccess Generator fixes this redirection issue by default. If you are using the HashLocationStrategy and you want to use other features of the .htaccess generator, it doesn't harm to keep the fix.

Words of advice

  1. If you have access to the Apache server configuration I recommend you to insert the options there instead of using a .htaccess file. That way it improves the page speed because the server can apply the options and redirection right away. If you prefer the easy way, you can still use the .htaccess file.
  2. Always do an backup before changing your .htaccess file. If something went wrong, you can easily go back.

FAQ

  1. The pages of my angular app are not indexed by search engines (e.g. Google). Why?

    Your app and it's content is generated by Angular on the client-side. Search engines can only access the meta tags of the index.html file. If you want to create an SEO friendly Angular app, see Angular Universal.

  2. Something does not work with the .htaccess file. What can I do?

    Please have a look on Trouble Shooting. If you are sure that this is a bug feel free to create an issue on Github. Otherwise you can go to Discussion and post a comment.

  3. I like to make feature requests, how can I do it?

    If you have feature requests or suggestions, feel free to create an issue on Github. Please describe your request as detailed as possible.
